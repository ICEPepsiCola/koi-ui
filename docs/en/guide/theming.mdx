# Theming

Koi UI themes are CSS variables. Pick a built-in skin with `data-theme`, or override tokens in your app.

## Built-in themes

Six skins ship in `@koi-ui/tokens`: `light`, `dark`, `ocean`, `forest`, `sunset`, `violet`.

```tsx preview
import { KoiProvider, Button, Alert, Tag, Space } from '@koi-ui/core';

export default () => (
  <KoiProvider theme={{ name: 'ocean' }}>
    <Space>
      <Button color="primary">Primary</Button>
      <Button color="primary" variant="soft">Soft</Button>
      <Button color="primary" variant="outline">Outline</Button>
      <Tag>Tag</Tag>
      <Alert title="Ocean theme" description="data-theme applied via KoiProvider" />
    </Space>
  </KoiProvider>
);
```

Docs demos also expose a theme switcher next to the desktop/mobile toggle.

## Provider overrides

`KoiProvider.theme` writes CSS variables on the theme root:

```tsx pure
import { KoiProvider } from '@koi-ui/core';

<KoiProvider theme={{ name: 'violet', primaryColor: 'hsl(280 80% 50%)', radiusMd: '1rem' }}>
  {children}
</KoiProvider>
```

## Manual CSS

```css
:root {
  --color-primary: hsl(160 84% 39%);
  --color-primary-foreground: hsl(0 0% 100%);
  --radius-md: 0.75rem;
}

html[data-theme='ocean'] {
  /* or any subtree */
}
```

## Dark mode

Built-in dark tokens follow `html.dark` / `html.rp-dark` / `data-theme="dark"`.

```css
html.dark {
  --color-surface: hsl(222 47% 11%);
  --color-surface-foreground: hsl(210 40% 98%);
}
```

`@koi-ui/core/styles.css` includes tokens by default. Tokens only:

```css
@import "@koi-ui/tokens/theme.css";
```

## Semantic colors

| Token | Role |
|-------|------|
| `--color-primary` | Brand |
| `--color-success` / `--color-warning` / `--color-info` / `--color-error` | Status |
| `--color-surface` / `--color-muted` / `--color-border` | Surfaces |
| `--radius-sm/md/lg` | Radius |

Component APIs use `color="error"` (daisyUI-aligned).

## Appearance language

Shared reading across controls:

- **solid / primary** — filled
- **soft** — tinted background
- **outline** — border only
- **ghost** — minimal / text

`Button` / `Alert` / `Tag` / `Badge` use `color` × `variant`. `NoticeBar` / `Progress` / `Toast` / `Notification` / `Result` use semantic color tokens.

## Breakpoint provider

`KoiProvider` can change the mobile breakpoint (default `lg` = 1024px):

```tsx preview
import { KoiProvider, Button } from '@koi-ui/core';

export default () => (
  <KoiProvider breakpoint="md">
    <Button>Breakpoint md (768px)</Button>
  </KoiProvider>
);
```
